
# 🔌 Projeto 4 – Testes de API

Este repositório contém o projeto desenvolvido na **4ª sprint**, com foco na criação e execução de **testes de API** para validação de requisições e respostas de serviços backend.

Os testes foram realizados utilizando o **Postman**, com o objetivo de garantir que os endpoints da aplicação funcionem corretamente, respeitando regras de negócio, formatos de dados e padrões de comunicação.

---

## 📌 Objetivo

O projeto tem como principal objetivo assegurar a qualidade dos serviços backend por meio de:

- Validação de endpoints da API  
- Verificação de regras de negócio  
- Garantia da integridade das respostas  
- Identificação e registro de falhas  

---

## 🛠️ Ferramentas Utilizadas

- **Postman** – Para envio de requisições e validação das respostas  
- **Jira** – Para rastreamento e gerenciamento de bugs encontrados  

---

## 🔍 Escopo dos Testes

Os testes cobrem diferentes aspectos das APIs, incluindo:

### 📡 Requisições HTTP

- Métodos: **GET, POST, PUT, DELETE**  
- Parâmetros de rota, query e body  

---

### 📦 Validação de Respostas

- Códigos de status HTTP (200, 201, 400, 404, etc.)  
- Estrutura e formato dos dados (JSON)  
- Campos obrigatórios e opcionais  
- Consistência das informações retornadas  

---

### ⚠️ Testes Negativos

- Requisições com dados inválidos  
- Campos obrigatórios ausentes  
- Validação de mensagens de erro  
- Comportamento da API em cenários inesperados  

---

## 🧪 Casos de Teste

Os casos de teste foram documentados de forma estruturada, contendo:

- ID do teste  
- Descrição  
- Pré-condições  
- Passos para execução (requisições)  
- Resultado esperado  
- Status (quando aplicável)  

---

## 🐞 Gerenciamento de Bugs

Todos os erros identificados durante os testes foram registrados no **Jira**, permitindo:

- Rastreabilidade dos defeitos  
- Priorização de correções  
- Acompanhamento do ciclo de vida dos bugs
